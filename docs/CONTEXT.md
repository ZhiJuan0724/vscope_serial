## 当前模块：Services/SerialService

### 已完成的决策
- 单例 `ChangeNotifier` 模式管理串口生命周期（2026-05-15）
- **Windows原生串口读取器**：C++ DLL通过`Dart_PostCObject_DL`回调，替代flutter_libserialport的读取功能（2026-05-20）
- **时间窗口聚合器**：按微秒粒度聚合零散数据，解决底层1B/3B/4B零散读取问题（2026-05-20）
- 原始数据双缓冲：512MB字节缓存（导出用）+ 500行文本缓存（显示用）（2026-05-20）
- 原始数据接收独立开关 `isRawReceiving`，与串口连接状态解耦（2026-05-20）
- **发送数据同步显示**：数据收发窗口同时显示接收和发送数据，HEX发送在文本显示下标记`[HEX]`（2026-05-21→2026-05-28调整）
- **显示分包策略**：仅 HEX显示 + 时间戳 开启时使用时间窗口聚合器；文本显示按`\r`/`\n`换行，不按底层回调分包（2026-05-28）
- CRC高低位反转选项（2026-05-21）
- 文本发送可选择自动在末尾追加`\r`、`\n`或`\r\n`，HEX发送不受影响（2026-05-29）
- 原始串口字节使用分块缓冲保存，避免长时间接收时 `List<int>` 扩容和头删成本（2026-05-22）
- 绘图时禁用逐包debug日志，避免性能瓶颈（2026-05-19）
- 断开串口自动关闭原始数据接收（2026-05-20）
- 日志统一使用`AppLogger`，移除所有`print()`（2026-05-21）
- **移除未使用的Isolate探测代码**：`_openPortInIsolate`、`_openPortInIsolateKeepOpen`、`_OpenPortArgs`、`_OpenPortResult`（2026-05-21）

### 已知约束
- `flutter_libserialport` **仅用于串口枚举**（`SerialPort.availablePorts`），读取/写入/配置已完全由 C++ DLL 替代（2026-05-22）
- `flutter_libserialport` 在Windows上虚拟串口（com0com）识别不稳定
- **C++ DLL关闭流程**：窗口关闭前需先断开串口，否则可能触发`abort()`（已通过`_WindowCloseListener`修复）
- 文本缓存按UTF-16估算大小，可能与实际内存占用有偏差
- 显示行数限制500行，超出后FIFO丢弃，导出时仍可从字节缓存还原完整数据
- 串口连接弹窗常用波特率从9600开始，并包含512000和1152000（2026-05-29）
- 串口配置输入变更立即写入持久化配置，不依赖连接成功后再保存（2026-05-29）

- `ChannelConfig` 默认16通道，每通道独立配置颜色/别名/可见性/偏移/缩放（2026-05-19）

- **ZobowConfigProfile**：配置文件模型，含名称+预设列表（2026-05-22）
- **ZobowChannelPreset**：键值对模型，名称+地址（2026-05-22）

- 众邦电控配置支持4/8通道，通道地址和类型列表固定归一到8项，当前生效数量由 `channelCount` 决定（2026-05-23）

- 众邦电控协议当前仅支持4/8通道两种固定帧长，不支持任意变长

## 当前模块：Data/Parsers

### 已完成的决策
- `IDataParser` 统一接口：`feed()` → `outputStream`（2026-05-15）
- FireWater：逗号分隔double，自动识别通道数或固定通道数（2026-05-15）
- 固定帧头：可配置1-4字节帧头、数据类型、通道数、可选校验和帧尾（2026-05-17）
- JustFloat：VOFA小端float32数组+`00 00 80 7F`帧尾，通道数为0时按帧长度自动识别，超过16通道或非4字节对齐视为异常（2026-05-29）
- **ZobowParser**（原JackFourChannelParser）：滑动窗口 + CRC16/MODBUS校验，无帧头（2026-05-18→2026-05-22重命名）
- ZobowParser 帧长随4/8通道配置动态变化，4通道为10字节，8通道为18字节，CRC算法保持CRC16/MODBUS（2026-05-23）

### 已知约束
- FireWater解析器不支持非ASCII字符（如中文）
- 固定帧头解析器校验和仅支持SUM8/SUM16，CRC校验待扩展
- 众邦电控解析器对丢包/错帧的恢复能力有限
